# ADR-001 PostgreSQL을 기본 데이터베이스로 선택

## 배경

- 우리 서비스는 사용자 행동 이벤트를 지속적으로 저장한다.
- 이후 organization, 시간, path 등 기준으로 데이터로 조회하고 집계한다.
- 따라서 데이터 정합성과 조건 기반 조회 및 집계 성능이 중요하다.

---

## 선택

프로젝트의 DB로 PostgreSQL을 사용하기로 결정했다.

---

## 이유

- 칼럼이 명확하게 정의된 정형 데이터 구조이기에 RDB가 적합하다.
- date_trunc와 같은 시간 기반 집계 함수를 제공한다.
  - 시간 단위 그룹핑을 SQL로 표현 가능하다.
- organization_id와 시간 조건 기반 필터링이 자주 필요하기에 복합 인덱스 전략이 유리하다.
