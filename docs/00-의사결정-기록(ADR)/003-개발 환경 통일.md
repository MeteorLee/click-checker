# ADR-003 개발 환경 통일

## 배경

- 배포 환경인 Ubuntu와 동일한 환경에서 프로젝트를 시작하고자 했다.
- Windows에 설치된 IntelliJ에서 작업할 경우 줄바꿈이 CRLF로 저장되었고, Ubuntu 기반 환경에서는 LF를 사용하여 Git diff가 불필요하게 발생했다.

---

## 선택

- 개발 환경을 Ubuntu 기반 (WSL)
- 실행 환경을 컨테이너 기반으로 통일 (Docker)
- 텍스트 줄바꿈을 LF로 고정 (.gitattributes)

---

## 이유

- 개발 환경과 배포 환경의 환경 차이를 줄여 오류가 발생할 가능성을 줄였다.
- Docker를 도입하여 실행 환경에서의 동일한 재현성을 확보했다.
- Git 오류를 예방하여 버전 관리의 신뢰성을 확보했다.
