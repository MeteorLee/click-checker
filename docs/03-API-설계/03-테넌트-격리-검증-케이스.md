# 테넌트 격리 검증 케이스 (2단계)

## 목적
- organization 스코프가 클라이언트 입력이 아닌 인증 컨텍스트로 강제되는지 확인한다.

## 필수 검증 케이스
1. 이벤트 저장: API Key 누락
- 요청: `POST /api/events` (헤더 `X-API-Key` 없음)
- 기대: `401 Unauthorized`

2. 이벤트 저장: API Key 무효
- 요청: `POST /api/events` + 무효 `X-API-Key`
- 기대: `401 Unauthorized`

3. 이벤트 저장: API Key 정상
- 요청: `POST /api/events` + 유효 `X-API-Key`
- 기대: `2xx`, 이벤트 저장 성공

4. 집계 조회: API Key 누락
- 요청: `GET /api/events/aggregates/*` (헤더 없음)
- 기대: `401 Unauthorized`

5. 집계 조회: API Key 무효
- 요청: `GET /api/events/aggregates/*` + 무효 `X-API-Key`
- 기대: `401 Unauthorized`

6. 집계 조회: API Key 정상
- 요청: `GET /api/events/aggregates/*` + 유효 `X-API-Key`
- 기대: `2xx`, 해당 organization 데이터만 조회

## 조직 위조 방지 검증
- `POST /api/events` 요청 body에 `organizationId`를 보내지 않는다.
- 조회 API 쿼리에서 `organizationId`를 사용하지 않는다.
- 조직 결정은 `X-API-Key -> authOrgId`로만 수행된다.
