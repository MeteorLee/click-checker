# 화면 흐름 (v1.0)

## 목표
- 1차 프런트 콘솔의 화면 흐름을 고정한다.
- 이번 단계에서 구현할 화면은 `로그인 -> organization 선택 -> overview 대시보드` 3개로 제한한다.
- 각 화면의 진입 조건, 이동 조건, API 호출 지점을 명확히 정리한다.

---

## 1. 전체 흐름

```text
/login
-> /organizations
-> /dashboard/[organizationId]
```

설명:
- 사용자는 먼저 로그인한다.
- 로그인 후 자신이 속한 organization 목록을 확인한다.
- organization을 하나 선택하면 해당 organization의 overview 대시보드로 이동한다.

---

## 2. 화면별 정의

## 2.1 로그인 화면

### 경로
- `/login`

### 목적
- 계정 로그인
- access token 확보

### 입력
- `loginId`
- `password`

### 호출 API
- `POST /api/v1/admin/auth/login`

### 성공 시
- access token 저장
- `/organizations`로 이동

### 실패 시
- 로그인 실패 메시지 표시
- 현재 화면 유지

### 화면 상태
- 기본 상태
- 제출 중 loading 상태
- 로그인 실패 에러 상태

---

## 2.2 organization 선택 화면

### 경로
- `/organizations`

### 목적
- 현재 로그인한 사용자가 접근 가능한 organization 목록 확인
- 대시보드 진입 organization 선택

### 진입 조건
- access token 존재

### 호출 API
- `GET /api/v1/admin/me`

### 화면 데이터
- account 정보
- memberships 목록
  - `organizationId`
  - `organizationName`
  - `role`

### 성공 시
- organization 목록 렌더링

### 사용자 액션
- organization 하나 선택

### 선택 후
- `/dashboard/[organizationId]`로 이동

### 실패 시
- `401`이면 `/login`으로 이동
- 그 외 에러는 에러 상태 표시

### 빈 상태
- membership이 하나도 없으면
  - "접근 가능한 organization이 없음" 상태 표시
  - 이후 조직 생성/초대 흐름은 후속 단계로 둔다

### 화면 상태
- loading 상태
- 목록 표시 상태
- 빈 상태
- 에러 상태

---

## 2.3 overview 대시보드 화면

### 경로
- `/dashboard/[organizationId]`

### 목적
- 선택된 organization의 overview 지표 표시

### 진입 조건
- access token 존재
- 선택된 `organizationId` 존재

### 호출 API
- `GET /api/v1/admin/organizations/{organizationId}/analytics/overview`

### 파라미터
- `from`
- `to`

설명:
- timezone 파라미터는 보내지 않는다.
- 날짜 해석은 백엔드 기준 `Asia/Seoul` 고정 정책을 따른다.

### 표시 정보
- organization 정보
- 기간 선택
- 총 이벤트 수
- 고유 사용자 수
- 식별 이벤트 비율
- 이전 기간 대비 비교
- top routes
- top event types

### 성공 시
- overview 카드와 요약 리스트 렌더링

### 실패 시
- `401`이면 `/login`으로 이동
- `403`이면 organization 접근 불가 상태 표시
- `404`이면 organization 없음 상태 표시
- 그 외는 일반 에러 상태 표시

### 화면 상태
- loading 상태
- 정상 데이터 상태
- 빈 결과 상태
- `403` 접근 불가 상태
- `404` organization 없음 상태
- 일반 에러 상태

---

## 3. 공통 이동 규칙

## 로그인 가드
- access token이 없으면 보호 화면에 직접 진입할 수 없다.
- `/organizations`, `/dashboard/[organizationId]` 진입 시 token이 없으면 `/login`으로 보낸다.

## 인증 만료 처리
- API 호출 중 `401`이 오면 access token이 유효하지 않은 것으로 보고 `/login`으로 보낸다.

## organization 선택 규칙
- overview 화면은 path의 `organizationId`를 기준으로 동작한다.
- URL에 임의의 `organizationId`를 넣더라도, 서버가 membership으로 다시 검증한다.

## 4. 1차 화면 구성 기준

## 로그인 화면
- 중앙 정렬된 로그인 카드
- 서비스명
- 입력 2개
- 로그인 버튼 1개

## organization 선택 화면
- 상단에 사용자 정보
- 본문에 organization 카드 목록
- 각 카드에 organization 이름 / role / 진입 버튼

## overview 대시보드 화면
- 상단 헤더
- 현재 organization 이름
- 기간 선택
- KPI 카드
- top routes 카드
- top event types 카드
