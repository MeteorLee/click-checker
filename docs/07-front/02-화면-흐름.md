# 화면 흐름 (v1.2)

## 목표
- 1차 프런트 콘솔의 실제 화면 흐름을 현재 구현 상태에 맞게 정리한다.
- 이번 단계에서 구현된 화면은 `회원가입/로그인 -> organization 선택/생성 -> overview 대시보드 -> routes / event types / trends` 흐름이다.
- organization 생성 직후 API key를 1회 표시하고, overview 대시보드 안에서 API key metadata / rotate를 관리하는 흐름까지 포함한다.

---

## 1. 전체 흐름

```text
/signup or /login
-> /organizations
-> organization 선택 또는 생성
-> 생성 시 API key 1회 표시
-> /dashboard/[organizationId]
-> 기간 전환 / organization 전환 / API key 관리
-> /dashboard/[organizationId]/routes
-> /dashboard/[organizationId]/event-types
-> /dashboard/[organizationId]/trends
```

설명:
- 사용자는 회원가입 또는 로그인으로 JWT를 확보한다.
- `/organizations`에서 기존 organization을 고르거나 새 organization을 만든다.
- 새 organization을 만들면 API key를 한 번 확인한 뒤 overview 대시보드로 들어간다.
- 대시보드에서는 overview 조회 외에도 기간 전환, organization 전환, API key metadata 조회와 rotate를 수행할 수 있다.
- overview에서 routes / event types / trends 상세 화면으로 이동할 수 있다.

## 2. 화면별 정의

## 2.1 회원가입 화면

### 경로
- `/signup`

### 목적
- 새 admin 계정 생성
- access token 확보

### 입력
- `loginId`
- `password`

### 호출 API
- `POST /api/v1/admin/auth/signup`

### 성공 시
- access token 저장
- `/organizations`로 이동

### 실패 시
- 회원가입 실패 메시지 표시
- 현재 화면 유지

## 2.3 organization 선택 화면

### 경로
- `/organizations`

### 목적
- 현재 로그인한 사용자가 접근 가능한 organization 목록 확인
- 새 organization 생성
- overview 대시보드 진입

### 진입 조건
- access token 존재

### 호출 API
- `GET /api/v1/admin/me`
- `POST /api/v1/admin/organizations`

### 화면 데이터
- `loginId`
- memberships 목록
  - `organizationId`
  - `organizationName`
  - `role`

### 사용자 액션
- 기존 organization 카드 선택
- 새 organization 이름 입력 후 생성

### organization 생성 성공 시
- 생성 응답의 `apiKey`, `apiKeyPrefix`를 모달로 1회 표시
- 복사 후 `/dashboard/[organizationId]`로 이동

### 실패 시
- `401`이면 `/login`으로 이동
- 그 외 에러는 에러 상태 표시

### 빈 상태
- membership이 하나도 없으면
  - "아직 접근 가능한 organization이 없음" 상태 표시
  - organization 생성 폼으로 첫 organization을 만들 수 있다

---

## 2.4 overview 대시보드 화면

### 경로
- `/dashboard/[organizationId]`

### 목적
- 선택된 organization의 overview 지표 표시
- organization 전환
- 기간 전환
- API key metadata 조회 / rotate

### 진입 조건
- access token 존재
- 선택된 `organizationId` 존재

### 호출 API
- `GET /api/v1/admin/me`
- `GET /api/v1/admin/organizations/{organizationId}/analytics/overview`
- `GET /api/v1/admin/organizations/{organizationId}/api-key`
- `POST /api/v1/admin/organizations/{organizationId}/api-key/rotate`

### 파라미터
- `from`
- `to`

설명:
- timezone 파라미터는 보내지 않는다.
- 날짜 해석은 백엔드 기준 `Asia/Seoul` 고정 정책을 따른다.
- 현재 preset은 `1일 / 7일 / 30일`이다.
- 현재 `1일`은 "최근 24시간"이 아니라 "오늘" 구간이다.

### 표시 정보
- organization 정보
- 기간 선택
- 총 이벤트 수
- 고유 사용자 수
- 식별 이벤트 비율
- 이전 기간 대비 비교
- top routes
- top event types
- API key prefix / status / lastUsedAt / rotatedAt
- trends 진입 CTA

### rotate 성공 시
- 새 API key를 모달로 1회 표시
- 복사 버튼 제공
- 이후에는 prefix / rotatedAt만 화면에 남는다

### 실패 시
- `401`이면 `/login`으로 이동
- `403`이면 organization 접근 불가 상태 표시
- `404`이면 organization 없음 상태 표시
- 그 외는 일반 에러 상태 표시

## 2.7 trends 상세 화면

### 경로
- `/dashboard/[organizationId]/trends`

### 목적
- 선택된 organization의 시계열 추이를 확인한다.

### 진입 경로
- overview 화면의 `시계열 추이 보기`

### 호출 API
- `GET /api/v1/admin/organizations/{organizationId}/analytics/trends`

### 파라미터
- `from`
- `to`
- `bucket`

### 화면 기능
- organization 전환
- 조회 단위 선택
  - `일 단위로 보기`
  - `시간 단위로 보기`
- 일 단위:
  - 기간 선택
- 시간 단위:
  - 날짜 하루 선택
- 이벤트 수 / 고유 사용자 수 선 그래프
- 현재 버킷 요약 표

### 규칙
- 전체 조회 기간은 최대 90일
- 시간 단위는 하루 전용
- 시간 단위로 전환하면 현재 종료일 기준 하루 범위로 자동 전환

## 3. 공통 이동 규칙

## 로그인 가드
- access token이 없으면 보호 화면에 직접 진입할 수 없다.
- `/organizations`, `/dashboard/[organizationId]`, `/dashboard/[organizationId]/routes`, `/dashboard/[organizationId]/event-types`, `/dashboard/[organizationId]/trends` 진입 시 token이 없으면 `/login`으로 보낸다.

## 인증 만료 처리
- API 호출 중 `401`이 오면 access token이 유효하지 않은 것으로 보고 `/login`으로 보낸다.

## organization 선택 규칙
- overview 화면은 path의 `organizationId`를 기준으로 동작한다.
- URL에 임의의 `organizationId`를 넣더라도, 서버가 membership으로 다시 검증한다.
- overview 화면 안에서도 organization select로 바로 다른 organization으로 이동할 수 있다.
- routes / event types / trends 화면도 같은 방식으로 organization select를 통해 바로 다른 organization으로 이동할 수 있다.
