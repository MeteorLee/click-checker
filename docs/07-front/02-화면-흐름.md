# 화면 흐름 (v1.5)

## 목표
- 현재 구현된 관리자 콘솔의 실제 화면 흐름을 정리한다.
- 이번 단계의 흐름은 `회원가입/로그인 -> organization 선택/생성 -> overview -> 상세 분석 / 설정 화면`이다.

---

## 1. 전체 흐름

```text
/signup or /login
-> /organizations
-> organization 선택 또는 생성
-> 생성 시 API key 1회 표시
-> /dashboard/[organizationId]
-> /dashboard/[organizationId]/routes
-> /dashboard/[organizationId]/event-types
-> /dashboard/[organizationId]/trends
-> /dashboard/[organizationId]/users
-> /dashboard/[organizationId]/activity
-> /dashboard/[organizationId]/retention
-> /dashboard/[organizationId]/funnels
-> /dashboard/[organizationId]/api-key
-> /dashboard/[organizationId]/route-templates
-> /dashboard/[organizationId]/event-type-mappings
-> /dashboard/[organizationId]/members
```

설명:
- 사용자는 회원가입 또는 로그인으로 JWT를 확보한다.
- `/organizations`에서 기존 organization을 고르거나 새 organization을 만든다.
- 새 organization을 만들면 API key를 한 번 확인한 뒤 overview 대시보드로 들어간다.
- overview에서 analytics 상세 화면과 설정/운영 화면으로 이동한다.

---

## 2. 화면별 정의

### 2.1 회원가입 화면
- 경로: `/signup`
- 목적: 새 admin 계정 생성, access token 확보
- 호출 API: `POST /api/v1/admin/auth/signup`

### 2.2 로그인 화면
- 경로: `/login`
- 목적: 기존 admin 계정 로그인, access token 확보
- 호출 API: `POST /api/v1/admin/auth/login`

### 2.3 organization 선택 화면
- 경로: `/organizations`
- 목적:
  - 현재 로그인한 사용자의 organization 목록 표시
  - organization 생성
  - organization 삭제 UI 제공
- 호출 API:
  - `GET /api/v1/admin/me`
  - `POST /api/v1/admin/organizations`
  - `DELETE /api/v1/admin/organizations/{organizationId}/members/membership`
- 삭제 규칙:
  - UI 문구는 `삭제`
  - 실제 동작은 membership 해제
  - 마지막 OWNER인데 다른 멤버가 남아 있으면 삭제 불가
  - 혼자 남은 OWNER이자 유일한 멤버면 확인 문구 입력 후 삭제 가능

### 2.4 overview 대시보드
- 경로: `/dashboard/[organizationId]`
- 목적:
  - overview 조회
  - analytics 허브
  - 설정/운영 허브
  - 추가 분석 / 설정 진입 허브
- 호출 API:
  - `GET /api/v1/admin/me`
  - `GET /api/v1/admin/organizations/{organizationId}/analytics/overview`
 - 구조:
  - 핵심 데이터
  - 추가 분석
  - 설정

### 2.5 routes
- 경로: `/dashboard/[organizationId]/routes`
- 목적: 정규화된 route 집계 상세 조회
- 추가 진입: route 규칙 관리

### 2.6 event types
- 경로: `/dashboard/[organizationId]/event-types`
- 목적: canonical event type 집계 상세 조회
- 추가 진입: event type 규칙 관리

### 2.7 trends
- 경로: `/dashboard/[organizationId]/trends`
- 목적: 시계열 추이 조회
- 규칙:
  - `일 단위로 보기`
  - `시간 단위로 보기`
  - 시간 단위는 하루 전용

### 2.8 users
- 경로: `/dashboard/[organizationId]/users`
- 목적: 현재 기간 사용자 현황 요약

### 2.9 activity
- 경로: `/dashboard/[organizationId]/activity`
- 목적: 기간 내 활동량과 분포 확인

### 2.10 retention
- 경로: `/dashboard/[organizationId]/retention`
- 목적: 기준일 코호트 기준 유지율 확인
- 규칙:
  - `정확히 N일 뒤`가 아니라 `N일 내 재방문`

### 2.11 funnels
- 경로: `/dashboard/[organizationId]/funnels`
- 목적: 단계별 전환율과 이탈 구간 확인
- 규칙:
  - 조회 기간과 전환 허용 기간은 구분
  - canonical event type 중심 단계 구성
  - routeKey는 세부 조건

### 2.12 route 규칙 관리
- 경로: `/dashboard/[organizationId]/route-templates`
- 목적: route template 조회 / 추가 / 수정 / 활성 / 삭제

### 2.13 API key 관리
- 경로: `/dashboard/[organizationId]/api-key`
- 목적:
  - API key metadata 조회
  - rotate
  - rotate 직후 새 key 1회 확인

### 2.14 event type 규칙 관리
- 경로: `/dashboard/[organizationId]/event-type-mappings`
- 목적: event type 매핑 조회 / 추가 / 수정 / 활성 / 삭제

### 2.15 멤버 관리
- 경로: `/dashboard/[organizationId]/members`
- 목적:
  - 기존 계정을 loginId 기준으로 멤버 추가
  - 역할 변경
  - 멤버 제거
- 규칙:
  - 토큰 기반 초대 수락은 없음
  - 기존 계정을 바로 membership에 연결

## 3. 공통 이동 규칙

### 로그인 가드
- access token이 없으면 보호 화면에 직접 진입할 수 없다.
- `/organizations` 이하 화면 진입 시 token이 없으면 `/login`으로 보낸다.

### 인증 만료 처리
- API 호출 중 `401`이 오면 `/login`으로 보낸다.

### organization 선택 규칙
- dashboard 이하 화면은 path의 `organizationId`를 기준으로 동작한다.
- organization 변경은 `/organizations`에서 다시 선택하는 흐름으로 단순화했다.

### dashboard 공통 탐색
- dashboard 하위 화면은 좌측 사이드바를 공통 탐색으로 사용한다.
- 사이드바는 `Analytics`와 `Settings`를 분리한다.
- analytics 전역 메뉴는 `overview / trends / users / activity / retention / funnels`를 포함한다.
- routes / event types는 overview 요약 카드에서 진입한다.

## 4. 현재 구현된 화면 구성

### 회원가입 / 로그인
- 좌우 분할 레이아웃
- 설명 패널 + 폼 카드

### organization 선택
- 상단 헤더
- organization 생성 카드
- membership 목록 카드
- 생성 직후 API key 모달
- membership 해제(삭제) 모달

### overview
- 상단 헤더
- 기간 preset / custom 날짜
- KPI 카드
- 추가 분석 섹션
- top routes / top event types
- 설정 섹션

### 상세 분석 / 설정 화면
- 상단 헤더 + 좌측 사이드바
- 기간 또는 단위 선택
- 표/차트
- 필요 시 운영 관리 화면 진입 버튼
