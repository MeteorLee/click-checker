# 🚀 로드맵

이 프로젝트는 단순 기능 구현이 아니라  **운영을 가정한 SaaS 구조 확장 과정**을 보여주는 것을 목표로 한다.

기능 → 배포 → 관측 → 안정성 → 확장 순으로 단계적으로 발전시켰다.
  
---  

## Observability First

> “측정 가능한 시스템”을 먼저 만든다.

- Docker 기반 실행 환경 구축
- Spring Actuator를 통한 health 기준 확립
- 구조화 로그 + requestId 기반 요청 추적
- Micrometer → Prometheus → Grafana 연결
- k6 기반 최소 부하 테스트 도입
- (선택) Sentry 에러 트래킹 연계

🎯 결과  기능 이전에 **측정/추적/재현 가능한 상태**를 확보한다.
  
---  

## Deployment Evolution

> 수동 → 자동 → 무중단 순으로 배포 전략을 확장한다.

- 수동 EC2 배포
- GitHub Actions 기반 자동 배포
- EC2 주도 Pull 배포 구조 실험
- 2대 서버 기반 무중단 배포 구성

🎯 결과  배포 구조별 차이와 트레이드오프를 이해하고  운영 안정성을 점진적으로 확보한다.
  
---  

## Production Hardening

> 운영 환경을 고려한 안정성 강화

- API Key 기반 인증 구조 도입
- Flyway 기반 스키마 관리
- PostgreSQL → RDS 분리
- Actuator 노출 최소화 및 보안 정책 강화

🎯 결과  개발 환경과 운영 환경을 명확히 분리하고  데이터/보안 안정성을 확보한다.
  
---  

## Service Expansion

> 서비스 자체의 기능적 확장

- 집계 API 고도화
- 요금제(Plan / Subscription) 도입
- 관리 콘솔 구축
- 멀티테넌트 구조 강화

🎯 결과  B2B SaaS 구조를 완성한다.
