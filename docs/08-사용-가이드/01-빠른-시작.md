# 빠른 시작

Click-Checker는 두 가지 방식으로 사용합니다.

- 관리자 콘솔: 로그인 후 브라우저에서 조직과 분석 화면을 관리합니다.
- 제품 API: `X-API-Key`로 이벤트를 적재하고 분석 API를 직접 호출합니다.

이 문서는 가장 짧은 흐름만 다룹니다.

1. 회원가입 또는 로그인
2. organization 생성
3. API key 확인
4. 이벤트 1건 전송
5. 분석 결과 확인

---

## 1. 로그인 후 organization 만들기

1. 브라우저에서 `/signup` 또는 `/login`으로 들어갑니다.
2. 로그인 후 `/organizations`에서 새 organization을 만듭니다.
3. 생성 직후 표시되는 API key를 복사합니다.

주의:
- API key 평문은 생성/재발급 시점에만 볼 수 있습니다.
- 브라우저 인증에는 API key가 아니라 JWT를 사용합니다.

---

## 2. 이벤트 1건 보내기

가장 간단한 예시는 아래와 같습니다.

```bash
curl -X POST http://localhost:8080/api/events \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <API_KEY>" \
  -d '{
    "eventType": "page_view",
    "path": "/",
    "externalUserId": "quick-start-user-1",
    "occurredAt": "2026-03-28T12:00:00Z"
  }'
```

설명:
- `eventType`: 원본 이벤트 이름입니다.
- `path`: 경로입니다.
- `externalUserId`: 있으면 식별 사용자 이벤트, 없으면 익명 이벤트입니다.
- `occurredAt`: 이벤트 발생 시각입니다.

---

## 3. overview 확인하기

이벤트를 보낸 뒤 overview API로 바로 확인할 수 있습니다.

```bash
curl "http://localhost:8080/api/v1/events/analytics/aggregates/overview?from=2026-03-28T00:00:00Z&to=2026-03-29T00:00:00Z" \
  -H "X-API-Key: <API_KEY>"
```

주요 응답 항목:
- `totalEvents`
- `uniqueUsers`
- `identifiedEventRate`
- `topRoutes`
- `topEventTypes`

overview는 짧은 확인용 요약에 가깝습니다.

---

## 4. activity 확인하기

더 분포 중심으로 보려면 `activity` API를 사용합니다.

```bash
curl "http://localhost:8080/api/v1/events/analytics/activity?from=2026-03-01T00:00:00Z&to=2026-03-29T00:00:00Z&timezone=Asia/Seoul" \
  -H "X-API-Key: <API_KEY>"
```

주요 응답 항목:
- `weekdaySummary`
- `weekendSummary`
- `dayOfWeekDistribution`
- `hourlyDistribution`
- `weekdayHourlyDistribution`
- `weekendHourlyDistribution`

이 API는 “언제 변화했는가”보다 “어느 요일/시간대에 몰리는가”를 보는 데 적합합니다.

---

## 5. 브라우저에서 확인하기

로그인 상태라면 콘솔에서도 같은 조직을 바로 볼 수 있습니다.

- `/organizations`
- `/dashboard/[organizationId]`
- `/dashboard/[organizationId]/activity`
- `/dashboard/[organizationId]/users`
- `/dashboard/[organizationId]/retention`
- `/dashboard/[organizationId]/funnels`

즉 제품 API로 이벤트를 보내고, 관리자 콘솔에서 시각적으로 확인하는 흐름이 기본입니다.

---

## 6. 관리자 콘솔 API와 제품 API의 차이

### 관리자 콘솔 API
- 인증: `Authorization: Bearer <accessToken>`
- 경로: `/api/v1/admin/**`
- 용도:
  - 로그인/회원가입
  - organization 생성
  - API key 관리
  - 멤버/규칙 관리
  - 콘솔용 analytics

### 제품 API
- 인증: `X-API-Key`
- 경로:
  - `POST /api/events`
  - `GET /api/v1/events/analytics/**`
- 용도:
  - 이벤트 적재
  - API key 기준 조직 분석 조회
