# 16. 웹 진입점 종합

## 0. 목표
- EC2에서 직접 노출하던 앱 진입 구조를 Nginx 기준으로 정리한다.
- `clickchecker.dev` 도메인과 HTTPS를 붙여 운영형 진입점을 만든다.
- 외부 공개 경로와 내부 관리 경로를 분리한다.
- Grafana도 메인 앱과 분리된 관리자 접근 경로로 정리한다.

---

## 1. 이번에 반영한 내용

### 1.1 도메인
- `clickchecker.dev` 구매
- `grafana.clickchecker.dev` DNS 추가
- DNS 관리 위치를 Route 53으로 전환
- `A` 레코드로 EC2 퍼블릭 IP 연결

### 1.2 Nginx
- EC2에 Nginx 설치
- `80/443 -> Nginx -> 127.0.0.1:8080` reverse proxy 구성
- `server_name clickchecker.dev` 적용
- `grafana.clickchecker.dev -> 127.0.0.1:3000` reverse proxy 구성
- Grafana 서브도메인에 nginx Basic Auth 적용

### 1.3 HTTPS
- `certbot --nginx -d clickchecker.dev`로 인증서 발급
- `grafana.clickchecker.dev`까지 포함하도록 인증서 확장
- Let's Encrypt 인증서 적용
- `80 -> 443` 리다이렉트 반영

### 1.4 공개/차단 정책
- `GET /` : 기본 응답 제공
- `/actuator/health` : 외부 공개
- `/actuator/prometheus` 포함 나머지 `/actuator/**` : 외부 차단
- app `8080` 직접 외부 노출 제거
- Grafana는 `grafana.clickchecker.dev + HTTPS + Basic Auth + Grafana 로그인` 기준으로 접근

---

## 2. 최종 구조

사용자  
→ `https://clickchecker.dev`  
→ Nginx (`80/443`)  
→ Spring app (`127.0.0.1:8080`)  
→ RDS

관리자  
→ `https://grafana.clickchecker.dev`  
→ Nginx Basic Auth  
→ Grafana 로그인  
→ Grafana (`127.0.0.1:3000`)

---

## 3. 확인 결과

### 3.1 루트 응답
- `curl -i -H 'Host: clickchecker.dev' http://127.0.0.1`
- 결과: `200`
- 응답:

```json
{"service":"click-checker","status":"ok","health":"/actuator/health"}
```

### 3.2 HTTPS 루트 응답
- `curl -I https://clickchecker.dev`
- 결과: `200`

### 3.3 Health 공개 확인
- `curl -I https://clickchecker.dev/actuator/health`
- 결과: `200`

### 3.4 내부 actuator 차단 확인
- `curl -I https://clickchecker.dev/actuator/prometheus`
- 결과: `403`

### 3.5 Grafana 접근 확인
- `https://grafana.clickchecker.dev`
- 결과: Basic Auth 선행 후 Grafana 로그인 화면 접근 가능

## 4. 현재 운영 상태
- 앱은 RDS를 사용한다.
- prod 배포 경로는 local `postgres` 컨테이너에 의존하지 않는다.
- 외부 진입점은 `clickchecker.dev` + HTTPS 기준으로 정리됐다.
- app `8080`은 더 이상 외부에서 직접 접근하지 않는다.

### 4.1 외부 공개 대상
- `https://clickchecker.dev`
- `https://clickchecker.dev/actuator/health`

### 4.2 제한 공개 대상
- `https://grafana.clickchecker.dev`
  - nginx Basic Auth 선행
  - Grafana 로그인 필요
  - 관리자 전용 제한 공개 상태

### 4.3 외부 비공개 대상
- `8080` app 직접 접근
- `5432` DB 접근
- `/actuator/prometheus`
- 기타 `/actuator/**`
- Prometheus 직접 접근 경로

---

## 5. 남은 후속 작업
- certbot 자동 갱신 점검
  - `sudo certbot renew --dry-run`
- Grafana 접근 제어를 Basic Auth보다 더 강한 방식으로 확장할지 검토

### 5.1 현재 미사용 정책
- `www.clickchecker.dev`는 현재 사용하지 않는다.
- 메인 서비스 진입점은 `clickchecker.dev` 하나로 유지한다.

## 7. 결론
- 웹 진입점은 이제 `EC2 퍼블릭 포트 직접 접근`이 아니라 `Nginx + 도메인 + HTTPS` 기준으로 동작한다.
- 운영 확인 경로는 `/actuator/health`로 유지하고, 내부 actuator는 외부에서 차단한다.
- app `8080` 직접 외부 노출도 제거되어, 외부 사용자는 nginx 진입점만 사용한다.
- Grafana도 `grafana.clickchecker.dev + HTTPS + Basic Auth + 로그인` 기준으로 관리자 접근 경로를 갖는다.
- 현재 상태는 포트폴리오 기준으로도 설명 가능한 운영형 진입점이다.
