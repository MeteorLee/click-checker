# 운영 진입점 정의 (1.X)

## 현재 기준(임시)
- 외부 클라이언트는 `HTTPS + organizationId` 기반으로 `POST /api/events`, `GET /api/events/aggregates/*`에 접근하며, 서버는 organization scope로 데이터 격리를 수행한다.

## 임시 정책 종료 조건
- 2단계(API Key 기반 인증) 적용 완료 시점에 `organizationId` 직접 전달 방식은 종료한다.
- 종료 완료 기준:
  1. 수집 API(`POST /api/events`)가 `X-API-Key` 없으면 `401`을 반환한다.
  2. 조회 API(`GET /api/events/aggregates/*`)도 `X-API-Key`로 organization을 확정한다.
  3. 외부 API 계약에서 `organizationId` 파라미터는 제거(또는 호환 기간 후 제거)한다.
