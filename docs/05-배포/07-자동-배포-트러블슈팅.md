# 07 자동 배포 트러블 슈팅 기록

## 보안 그룹 ssh 추가

- 증상:
    - GitHub Actions가 EC2에 SSH로 접근하지 못함
    - EC2 보안 그룹의 ssh가 내 개인 IP만 허용된 상태
- 임시 대응:
    - 일단 임시로 보안 그룹에서 `0.0.0.0/0`으로 모든 IP에서 ssh 22포트 접근이 가능하도록 열어둠
- 향후 대응:
    - CodeDeploy + Pull 방식을 도입할 예정

## Health check 리트라이 추가

- 증상:
    - Postgres가 정상 작동하지만 app health check만 통과하지 못함
- 원인:
    - Docker 빌드 과정에서 DB인 PostgreSQL이 늦게 뜨는 상황에서 health check가 발생하며 health check를 통과하지 못하는 상황이 발생
- 대응:
    - health check 리트라이 로직을 추가하여 대응

## 스모크 테스트 데이터 설정

- 증상:
    - 스모크 테스트 상황에서 테스트 데이터가 실제 DB의 데이터와 섞이며 데이터 오염이 발생
- 대응:
    - 스모크 테스트 데이터에 `__smoke` 태그를 붙여 필터링을 통한 삭제가 용이하도록 함
- 향후 계획
    - 스모크 테스트 데이터는 자동 정리 메커니즘
    - 스모크 전용 테넌트 도입

## 스모크 테스트 오류

- 증상:
    -  prod 배포 워크플로우에서 `EVENT_CODE=500`으로 실패
    - EC2에서 curl 테스트 상황에서는 정상(200)
- 임시 대응:
    - JSON 파싱, 바인딩, ExceptionHandler 구현
    - 배포 워크플로우 단계에서 에러 상황을 명확히 출력하도록 로그 추가
- 원인:
    - 로그를 통해 `appleboy/ssh-action` 환경에서 `jq` JSON 파싱 과정의 멀티라인 쉘 파싱 오류 발생을 확인
- 대응:
    - jq 파싱 코드를 멀티라인 → 원라인 수정

## CI 세분화 수정

- 증상:
    - main에서 push, pull request 중복 검증 발생
    - develop, prod에서 필요 없는 상황에서 ci를 통한 검증 발생
- 대응:
    - develop: push 상황에서만 빠른 h2 테스트
    - main: PR 상황에서만 엄격한 테스트
    - prod:
        - 배포를 위한 health check / 스모크 테스트
        - docker를 통한 이미지 생성 및 배포 + 실패 시 rollback
