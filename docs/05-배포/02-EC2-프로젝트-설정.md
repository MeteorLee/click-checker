# EC2 프로젝트 설정 (1.2)

## 0. 목적

- Click-Checker를 EC2(Ubuntu)에서 운영/배포하기 위한 서버 기본 설정을 1회 정리한다.
- 수동 배포, 자동 배포(deploy.sh, GitHub Actions) 모두 이 설정을 전제로 한다.
- 이 문서는 “서버 토대 문서”이며, 배포 절차 문서와 분리한다.

---

# 1. 기준 환경

## 1.1 OS

- Ubuntu 22.04 이상 권장

## 1.2 접속 사용자

- 기본 사용자: ubuntu
- SSH 키(.pem) 기반 접속

PEM 키 권한 설정 (로컬 PC):

```bash
chmod 400 ~/click-checker.pem
```

접속 예시:

```bash
ssh -i ~/click-checker.pem ubuntu@<EC2_PUBLIC_IP>
```

---

# 2. 보안 그룹 원칙

v1 단계에서는 직접 포트 확인을 허용하되, 반드시 “내 IP만” 허용한다.

- 22 (SSH): 내 IP
- 8080 (앱 확인용): 내 IP
- 3000 (Grafana 확인용): 내 IP

운영 확장 시:

- 80 / 443은 리버스 프록시(Nginx 등)에서 처리 권장
- 내부 서비스(DB 등)는 외부 포트 미노출 원칙

# 3. 필수 설치 항목

## 3.1 기본 패키지

- ca-certificates (HTTPS 통신용 인증서)
- curl
- git
- zip
- unzip
- jq (JSON 파싱용)

설치:

```bash
sudo apt-get update  
sudo apt-get install -y ca-certificates curl git zip unzip jq
```

---

## 3.2 Java (JDK 21)

```bash
sudo apt-get install -y openjdk-21-jdk  
java -version
```
---

## 3.3 Docker / Docker Compose (필수)

설치:

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER  
newgrp docker
```

확인:

```bash
docker --version  
docker compose version
```

# 4. 프로젝트 디렉토리 구조

## 4.1 프로젝트 경로 (고정 권장)

- ~/click-checker

자동 배포 및 스크립트는 이 경로를 기준으로 동작한다.

## 4.2 최초 클론

```bash
cd ~  
git clone <YOUR_REPO_URL> click-checker  
cd click-checker
```

---

# 5. 환경 변수 관리 원칙

## 5.1 .env 파일

- 위치: 프로젝트 루트 (~/click-checker/.env)
- Git에는 절대 포함하지 않는다.
- 서버에서만 관리한다.

생성:

```bash
cd ~/click-checker  
nano .env
```

예시:

```
POSTGRES_DB=click_checker  
POSTGRES_USER=app  
POSTGRES_PASSWORD=apppw  
SENTRY_DSN=
```

저장:
- Ctrl + O
- Enter
- Ctrl + X

원칙:

- .env는 서버 고정 설정이다.
- 자동 배포는 .env를 수정하지 않는다.
- 비밀 값은 GitHub에 올리지 않는다.

---

# 6. Docker Compose 구성 원칙

## 6.1 사용 파일

- docker-compose.yml
- docker-compose.prod.yml

## 6.2 네트워크 원칙

- app ↔ postgres ↔ prometheus ↔ grafana 는 내부 네트워크로 통신
- DB는 외부 포트 미노출 권장
- Grafana는 v1 단계에서만 내 IP로 노출

---

# 7. 서버 점검 기본 명령

## 7.1 컨테이너 상태 확인

```bash
cd ~/click-checker  
docker compose -f docker-compose.yml -f docker-compose.prod.yml ps
```

## 7.2 로그 확인

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml logs --tail=200 app  
docker compose -f docker-compose.yml -f docker-compose.prod.yml logs --tail=200 postgres
```
