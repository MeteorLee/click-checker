# EC2 프로젝트 설정 (1.4)

## 0. 목적

- Click-Checker를 EC2(Ubuntu)에서 운영/배포하기 위한 서버 기본 설정을 1회 정리한다.
- 수동 배포, 자동 배포(deploy.sh, GitHub Actions) 모두 이 설정을 전제로 한다.
- 이 문서는 “서버 토대 문서”이며, 배포 절차 문서와 분리한다.

---

# 1. 기준 환경

## 1.1 OS

- Ubuntu 22.04 이상 권장

## 1.2 접속 사용자

- 기본 사용자: ubuntu
- SSH 키(.pem) 기반 접속

PEM 키 권한 설정 (로컬 PC):

```bash
chmod 400 ~/click-checker.pem
```

접속 예시:

```bash
ssh -i ~/click-checker.pem ubuntu@<EC2_PUBLIC_IP>
```

---

# 2. 보안 그룹 원칙

현재 운영 기준:

- 22 (SSH): 내 IP
- 80 (HTTP): 외부 공개
- 443 (HTTPS): 외부 공개

원칙:

- 앱 `8080`은 외부 직접 노출하지 않는다.
- DB (`5432`)는 외부에 노출하지 않는다.
- Grafana `3000`도 외부 직접 노출하지 않는다.
- 내부 서비스는 가능한 한 Nginx 뒤로 숨긴다.
- 운영 진입점은 `clickchecker.dev`, `grafana.clickchecker.dev`와 Nginx가 담당한다.

# 3. 필수 설치 항목

## 3.1 기본 패키지

- ca-certificates (HTTPS 통신용 인증서)
- curl
- git
- zip
- unzip
- jq (JSON 파싱용)
- nginx

설치:

```bash
sudo apt-get update  
sudo apt-get install -y ca-certificates curl git zip unzip jq nginx
```

---

## 3.2 Java (JDK 21)

```bash
sudo apt-get install -y openjdk-21-jdk  
java -version
```
---

## 3.3 Docker / Docker Compose (필수)

설치:

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER  
newgrp docker
```

확인:

```bash
docker --version  
docker compose version
```

---

## 3.4 HTTPS / certbot

- 도메인: `clickchecker.dev`
- Grafana 도메인: `grafana.clickchecker.dev`
- 인증서 발급 도구: `certbot`
- nginx 연동 플러그인: `python3-certbot-nginx`

설치:

```bash
sudo apt-get update
sudo apt-get install -y certbot python3-certbot-nginx
```

인증서 발급:

```bash
sudo certbot --nginx -d clickchecker.dev -d grafana.clickchecker.dev
```

자동 갱신 점검:

```bash
sudo certbot renew --dry-run
```

# 5. 환경 변수 관리 원칙

## 5.1 .env 파일

- 위치: 프로젝트 루트 (~/click-checker/.env)
- Git에는 절대 포함하지 않는다.
- 서버에서만 관리한다.

생성:

```bash
cd ~/click-checker  
nano .env
```

예시:

```
POSTGRES_DB=click_checker  
POSTGRES_USER=app  
POSTGRES_PASSWORD=apppw  

DB_URL=jdbc:postgresql://<RDS_ENDPOINT>:5432/click_checker
DB_USERNAME=app
DB_PASSWORD=<RDS_PASSWORD>

API_KEY_PEPPER=<API_KEY_PEPPER>
API_KEY_ENV=live

SENTRY_DSN=
```

저장:
- Ctrl + O
- Enter
- Ctrl + X

원칙:

- .env는 서버 고정 설정이다.
- 자동 배포는 .env를 수정하지 않는다.
- 비밀 값은 GitHub에 올리지 않는다.
- `POSTGRES_*`는 로컬 postgres 컨테이너용, `DB_*`는 prod 앱이 실제로 접속할 DB용이다.
- prod 앱은 EC2 내부 postgres가 아니라 RDS를 기준으로 동작한다.
- `API_KEY_PEPPER`, `API_KEY_ENV`도 운영 필수값으로 함께 관리한다.

# 6. Docker Compose 구성 원칙

## 6.1 사용 파일

- docker-compose.yml
- docker-compose.prod.yml
- nginx/click-checker.conf

## 6.2 네트워크 원칙

- local: `app ↔ postgres ↔ prometheus ↔ grafana` 내부 네트워크 사용
- prod: `app ↔ RDS`, `app ↔ prometheus ↔ grafana` 구조 사용
- 외부 사용자는 `https://clickchecker.dev`로만 app에 접근
- Grafana 관리자는 `https://grafana.clickchecker.dev`로 접근
- app `8080`은 외부 직접 노출하지 않음
- DB는 외부 포트 미노출
- Grafana `3000`은 외부 직접 노출하지 않음
- Grafana는 nginx Basic Auth + Grafana 로그인 기준으로 접근 제어
- RDS는 EC2와 같은 VPC / 보안 그룹 기준으로 접근 제어한다.

## 6.3 nginx 설정 반영

repo 기준 설정 파일:

```bash
~/click-checker/nginx/click-checker.conf
```

실제 nginx 반영:

```bash
sudo cp ~/click-checker/nginx/click-checker.conf /etc/nginx/sites-available/default
sudo nginx -t
sudo systemctl reload nginx
```

# 7. 서버 점검 기본 명령

## 7.1 컨테이너 상태 확인

```bash
cd ~/click-checker  
docker compose -f docker-compose.yml -f docker-compose.prod.yml ps
```

## 7.2 로그 확인

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml logs --tail=200 app  
```

참고:
- local 디버깅 시에만 `docker compose ... logs --tail=200 postgres`를 본다.
- prod 운영 경로는 RDS 기준이라 postgres 로그를 기본 점검 대상으로 보지 않는다.
