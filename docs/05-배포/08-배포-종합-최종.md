# 자동 배포 종합 문서

## 1. 문서 목적
- 현재 자동 배포와 운영 기준을 한 문서로 정리한다.

---  

## 2. 현재 배포 범위 (v1)
- 인프라: 단일 EC2
- 실행: Docker Compose
- 애플리케이션: Spring Boot + PostgreSQL
- 관측: Prometheus + Grafana
- 자동 배포: GitHub Actions (`prod` push 트리거) + SSH
- 검증: Health check + Smoke test
- 실패 대응: 자동 롤백

---  

## 3. 브랜치/CI/CD 정책
### 3.1 브랜치 역할
- `develop`: 기능 개발 통합 브랜치
- `main`: 릴리즈 후보 검증 브랜치
- `prod`: 배포 전용 브랜치

### 3.2 CI 정책
- `ci-develop.yml`
  - 트리거: `develop` push
  - 검증: 빠른 테스트(H2)
- `ci-main.yml`
  - 트리거: `main` PR
  - 검증: `test` + `postgresTest`

### 3.3 CD 정책
- `deploy-prod.yml`
  - 트리거: `prod` push
  - 방식: EC2 SSH 접속 후 `docker compose up -d --build`
  - 검증:
    - `/actuator/health` 리트라이
    - Organization 생성 -> Event 저장 -> 집계 조회 스모크
  - 실패 시: 직전 SHA로 `git reset --hard` 후 재기동/재검증

---  

## 4. 배포 구성 파일 기준
- 기본: `docker-compose.yml`
  - app 8080 외부 공개
  - postgres/prometheus/grafana 기본 포트는 비공개(로컬 바인딩 없음)
- 로컬 오버레이: `docker-compose.local.yml`
  - postgres/prometheus/grafana 포트 오픈
- 운영 오버레이: `docker-compose.prod.yml`
  - app profile: `prod`
  - grafana `3000:3000` 오픈

실행 예시:
```bash  
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build app postgres prometheus grafana
```

## 5. 성공/실패 기준
### 5.1 배포 성공 기준
1. Health check 성공 (`UP`)
2. 스모크 쓰기 API 성공 (POST)
3. 스모크 읽기 API 성공 (GET)

### 5.2 배포 실패 기준
- health 리트라이 초과
- 스모크 API 실패(organization 생성/이벤트 저장/집계 조회 중 1개라도 실패)

### 5.3 롤백 기준
- 위 실패 조건 발생 시 즉시 롤백
- 롤백 후 health 재확인

---  

## 6. 스모크 테스트 정책 (현재)
- 스모크 데이터는 구분 가능한 태그를 사용한다.
  - organization name: `__smoke-org-<timestamp>`
  - eventType: `__smoke`
  - path: `/__smoke/deploy`
- 현재는 누적 허용
- 향후 개선:
  - 스모크 전용 tenant 도입
  - 정리 배치/TTL 정책 도입

---  

## 7. 주요 트러블슈팅 반영 사항
- app 부팅 지연 대응: health check 리트라이 추가
- smoke 이벤트 500 대응:
  - `jq` JSON 생성 구문 안정화(원라인)
  - 에러 시 응답 본문/컨테이너 로그 출력 강화
- EC2 SSH 접근 이슈:
  - 현재 SSH 기반 운영(임시)
  - 추후 배포 방식 전환 시 보안 정책 재정의

---  

## 8. 운영 시 체크 포인트
- 배포 전:
  - GitHub Secrets 설정 확인 (`EC2_HOST`, `EC2_USER`, `EC2_SSH_KEY`)
  - EC2 내 `jq` 설치 확인
  - `.env` 값 확인
- 배포 후:
  - `docker compose ... ps` 상태 확인
  - app/grafana 접근 확인
  - smoke 데이터 생성 여부 확인
