# 26. 배포 실전 검증 기록

## 문서 목적

배포 실전 검증 과정에서 실제로 실행한 baseline, 배포 중 검증, 결과 수치, 해석을 기록한다.  
이번 문서는 절차를 설명하는 런북이 아니라, 어떤 조건에서 무엇을 실행했고 어떤 수치가 나왔는지 남기는 데 목적이 있다.

## 1. 검증 범위

- 대상 환경: `prod`
- 검증 대상:
  - 지속 read 요청 baseline
  - 지속 read 요청 중 배포 전환
  - 지속 write 요청 baseline
  - 지속 write 요청 중 배포 전환
- 관찰 기준:
  - `http_req_failed`
  - timeout
  - `p95`
  - `p99`
  - `RPS`
  - `5xx`

## 2. Baseline - Read

### 2.1 실행 정보

- 실행일: `2026-03-12`
- 실행 위치: `local`
- 대상 URL: `https://clickchecker.dev`
- runLabel: `deploy-verify-local-20260312`
- 조직: `Acme Analytics deploy-verify-local-20260312`
- apiKeyPrefix: `468d7aec`
- 시나리오: `k6/smoke-read.js`
- 기간: `2026-03-01T00:00:00Z ~ 2026-03-06T00:00:00Z`
- 목표 RPS: `20 req/s`
- 실행 시간: `2m`

### 2.2 실행 명령

```bash
docker run --rm -i \
  -e API_KEY='<redacted>' \
  -e BASE_URL='https://clickchecker.dev' \
  -e FROM='2026-03-01T00:00:00Z' \
  -e TO='2026-03-06T00:00:00Z' \
  -e TOP='5' \
  -v "$PWD":/work -w /work \
  grafana/k6 run k6/smoke-read.js
```

### 2.3 결과

- total requests: `2401`
- `http_req_failed`: `0.00%`
- checks failed: `0`
- status `200` 유지
- `http_req_duration.avg`: `21.7ms`
- `http_req_duration.p95`: `25.93ms`
- `http_req_duration.p99`: `53.37ms`
- max observed duration: `1s`
- observed RPS: `19.122774/s`

### 2.4 해석

- 지속 read 요청 baseline에서는 실패 응답 없이 안정적으로 응답했다.
- `p95`, `p99`는 현재 목표 기준 안에 들어온다.
- 이후 배포 중 검증은 이 baseline과 비교해 지연 증가, `5xx`, timeout 유무를 본다.

## 6. 현재 메모

- 검증 준비와 `k6` 실행 위치는 `local`로 고정한다.
- EC2는 검증 스크립트 실행 위치가 아니라 배포 대상 서버로 본다.
- seed 결과 파일에는 plain `apiKey`가 들어가므로 보관과 노출 범위를 주의한다.
