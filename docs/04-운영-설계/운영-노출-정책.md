# 운영 노출 정책 (1.0)

## 1. 목적
- 외부에 공개할 대상과 내부 전용 대상을 명확히 구분한다.
- 운영 노출 정책을 앱 진입점, 관리 도구, 내부 포트 기준으로 고정한다.
- 이후 Grafana 도메인화나 접근 정책 변경 시 기준 문서로 사용한다.

---

## 2. 현재 기본 원칙
- 메인 서비스 진입점은 `clickchecker.dev` 하나로 운영한다.
- 외부 사용자는 Nginx를 통해서만 app에 접근한다.
- health 확인은 최소 범위로만 공개한다.
- 관리 도구와 내부 서비스는 메인 서비스와 분리해서 판단한다.

---

## 3. 외부 공개 대상

### 3.1 메인 서비스
- `https://clickchecker.dev`
- 역할:
  - 메인 서비스 진입점
  - 루트(`/`) 기본 응답 제공
  - API 요청 진입 경로

### 3.2 제한적 health 공개
- `https://clickchecker.dev/actuator/health`
- 역할:
  - 운영 기동 확인
  - 배포 후 health 검증 기준

---

## 4. 제한 공개 대상

### 4.1 Grafana
- 현재 접근 방식:
  - `https://grafana.clickchecker.dev`
  - nginx Basic Auth 선행
  - Grafana 로그인 필요
- 해석:
  - 외부 전체 공개는 아님
  - 관리자 전용 제한 공개 상태

### 4.2 현재 제한 기준
- 현재는 도메인 진입점이 생겼더라도 Grafana를 일반 사용자 기능으로 취급하지 않는다.
- 관리 도구 접근은 메인 서비스와 분리해서 유지한다.

---

## 5. 외부 비공개 대상

### 5.1 애플리케이션 내부 포트
- app `8080`
- 원칙:
  - Nginx 뒤의 내부 포트로만 사용
  - 외부 직접 접근 금지

### 5.2 DB 포트
- PostgreSQL / RDS `5432`
- 원칙:
  - 외부 공개 금지
  - EC2 또는 보안 그룹 기준 내부 접근만 허용

### 5.3 actuator 내부 경로
- `/actuator/prometheus`
- 기타 `/actuator/**`
- 원칙:
  - `/actuator/health` 외 경로는 외부 진입점에서 차단

### 5.4 Prometheus
- 직접 외부 공개하지 않음
- 별도 도메인도 만들지 않음
- 현재는 비공개 유지 정책으로 확정

---

## 6. 도메인 정책

### 6.1 현재 사용 도메인
- `clickchecker.dev`
- `grafana.clickchecker.dev`

### 6.2 현재 미사용 도메인
- `www.clickchecker.dev`
- 원칙:
  - 메인 서비스는 루트 도메인만 사용
  - `www`는 현재 지원하지 않음

### 6.3 후속 검토 대상
- `www.clickchecker.dev`
- 필요 시 Grafana 접근 제어를 Basic Auth에서 더 강한 방식으로 확장할지 검토

---

## 7. 정책 변경 기준
- 외부 공개 대상을 늘릴 때는 아래를 같이 판단한다.
  1. 실제 사용자 대상 기능인지
  2. 관리자 전용 도구인지
  3. 인증/접근제어가 충분한지
  4. 도메인화할 가치가 있는지

---

## 8. 현재 결론
- 외부 공개는 `clickchecker.dev`와 `/actuator/health`로 제한한다.
- Grafana는 `grafana.clickchecker.dev + HTTPS + Basic Auth + Grafana 로그인` 기준의 관리자 전용 제한 공개 상태로 유지한다.
- Prometheus는 별도 도메인 없이 외부 비공개를 유지한다.
- DB, app 내부 포트, 기타 actuator는 외부 비공개를 유지한다.
