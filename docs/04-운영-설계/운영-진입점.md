# 운영 진입점 정의 (2.0)

## 현재 기준
- 외부 클라이언트는 `HTTPS + X-API-Key` 기반으로 `POST /api/events`, `GET /api/v1/events/analytics/**`에 접근한다.
- 서버는 `X-API-Key` 인증 결과로 organization을 확정하며, 요청 body/query의 `organizationId`는 신뢰하지 않는다.
- 수집/조회 이벤트 API는 `X-API-Key` 누락/무효 시 `401`을 반환한다.
- 관리자 콘솔 API는 `JWT Bearer` 기반으로 `POST /api/v1/admin/auth/**`, `GET|POST|PATCH|DELETE /api/v1/admin/**`에 접근한다.
- 관리자 콘솔의 refresh 흐름은 `POST /api/v1/admin/auth/refresh` 기준으로 access token을 재발급한다.
- 외부 공개 범위와 포트/도메인 정책은 [운영-노출-정책.md](운영-노출-정책.md)를 기준으로 본다.

## 보호 범위(현재)
- 보호 대상:
  - `/api/events/**`
  - `/api/v1/events/**`
  - `/api/v1/admin/**` (`/api/v1/admin/auth/**` 제외)
- 비보호 대상(현재):
  - `/api/organizations/**`
  - `/api/v1/admin/auth/**`
- 주:
  - 비보호 대상은 후속 단계에서 보호 범위 확대 시 재검토한다.
  - `POST /api/organizations`는 기존 공개 organization 생성 경로이며, 현재 운영/포트폴리오 기준 주 경로는 `POST /api/v1/admin/organizations`다.
  - `POST /api/v1/admin/organizations/demo/join`은 보호 대상이며, 로그인한 계정을 seeded demo organization에 `VIEWER`로 연결하는 운영/데모 편의 경로다.

## 요청 추적 표준 (RequestId)
- 클라이언트가 `X-Request-Id`를 보내면 그대로 사용한다.
- 없으면 서버가 UUID를 생성한다.
- 응답 헤더 `X-Request-Id`로 동일 값을 반환한다.
- 서버 로그는 `requestId`(MDC)를 기준으로 요청 단위 추적을 수행한다.

## 운영 에러 분류 기준(요약)
- `401`: 인증 자격 누락/무효
  - API Key 누락/무효
  - JWT 누락/만료/무효
- `403`: 인증은 되었지만 조직 membership 또는 role 부족
- `404`: 없는 organization 또는 조회 대상 부재
- `400`: 요청 형식/파라미터 검증 오류
- `500`: 서버 내부 오류(DB/예상치 못한 예외)
